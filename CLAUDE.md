# CLAUDE.md — دليل الوكيل (اقرأه أولًا في كل جلسة)

هذا الملف يُقرأ تلقائيًا في بداية كل جلسة. اقرأه ثم اقرأ **`SESSIONS.md`** لمعرفة
الحالة الحالية والخطوة القادمة. لا تُعِد اكتشاف المشروع من الصفر.

## أسلوب الرد (تفضيل المالك)
- **بالعربية** دائمًا.
- **مختصر** بدون كلام هامشي أو استطراد.
- **لا تشرح الكود بعد إنجاز المهمة** إلا عند الطلب الصريح (توفيرًا للتوكنز).

## المشروع
بوابة نظام معلومات طلاب (SIS) ثنائية اللغة (عربي/إنجليزي، RTL).
- **التقنية**: Flask 3 + SQLite (وضع WAL). لا إطار أمامي — قوالب Jinja + CSS ثابت.
- **المعمارية**: طبقة خدمات — كل خدمة في `*_service.py` تأخذ `sqlite3.Connection`
  في مُنشئها (مثل `curriculum_service.py`, `grading_service.py`, `timetable_service.py`).
  المسارات في `webapp.py`، المخطط والترحيلات في `database.py`، النماذج في `models.py`.
- **اللغة (i18n)**: كل النصوص في `i18n.py` كقاموس `{key: {en, ar}}`، وتُستدعى في
  القوالب عبر `t()/ts()/tk()/tf()`. **ممنوع** كتابة نص ظاهر للمستخدم مباشرة في القالب.
- **الأدوار**: admin / registrar / teacher / accounting. الحماية عبر `staff_required(*roles)`
  و`portal_login_required`. CSRF إلزامي على كل POST. كلمات المرور PBKDF2.
- **الدرجات**: من 100 → حروف → معدل 5.0 موزون بالساعات؛ تقسيم أعمال/نهائي مرن لكل مقرر.
- **النشر**: PythonAnywhere (الطبقة المجانية: قرص 512MB، SQLite) — كن مقتصدًا في التخزين.

## قواعد صارمة (غير قابلة للتفاوض)
1. **ممنوع تغيير التصميم/الثيم** (ألوان، خطوط، تنسيق) — قرار نهائي من المالك. أعِد استخدام
   الأصناف الموجودة فقط: `card / table / stamp / btn / nav-menu / grid-2 / page-header`.
2. أي شك في متطلب = **لا تخمين**. سجّل السؤال في «أسئلة للمالك» بأسفل `SESSIONS.md`
   ونفّذ بقية العمل الواضح.
3. الجلسات المعلّقة (حاليًا 3 و5 و8) لا تُنفَّذ إلا بطلب صريح من المالك.
4. لا تُضِف تبعيات ثقيلة (مثل OR-Tools) — لن تعمل على الطبقة المجانية. بايثون خالص.

## الأوامر
- الاختبارات: `python -m pytest -q` (يجب أن تنجح كلها قبل أي دفع — حاليًا 73 اختبارًا).
- التشغيل محليًا: `python webapp.py` (المنفذ الافتراضي؛ `SIS_DEBUG=1` للتنقيح).
- بيانات تجريبية: `python seed_demo.py`.
- المتطلبات: `requirements.txt` (تشغيل) و`requirements-dev.txt` (تطوير/اختبار).

## سير العمل مع Git
- العمل على فرع **`main`** والدفع إليه مباشرة: `git push -u origin main`.
- عند فشل الدفع لأسباب شبكية: أعِد المحاولة حتى 4 مرات (2s, 4s, 8s, 16s).
- كل رسالة commit تنتهي بسطرَي التذييل القياسيَّين (المؤلف المشارك + رابط الجلسة).
- **ممنوع** وضع مُعرّف النموذج التقني داخل أي commit أو كود أو PR.

## روتين نهاية الجلسة (إلزامي — هذا ما يحفظ عملك)
قبل إنهاء أي دور أجريت فيه تعديلات، بالترتيب:
1. `python -m pytest -q` → كلها ناجحة.
2. جرّب الميزة فعليًا (متصفح/تشغيل) إن أمكن.
3. `git add -A && git commit` ثم `git push -u origin main`.
4. حدّث حالة الجلسة في `SESSIONS.md` (مكتملة / جارية + «توقفنا عند: …»).

الحاوية مؤقتة: أي عمل غير مدفوع إلى `main` **يُفقد** عند انتهاء الجلسة. الدفع = الحفظ.
